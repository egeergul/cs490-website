About the project

Software development is a crucial and core aspect of engineering. Millions of softwares are being developed each day with different sizes and complexities. From 1 to 100+ developers can work on the same project. As the size and the complexity of the software increases, by nature, the development process gets more challenging. Since efficiency is a core aspect of engineering, making the best use out of the limited resources (i.e. time) gains even more importance in such softwares. Many practices have proven that with simple ideas and some tools, software development can be facilitated ( i.e. version control systems). Thus, it is safe to say that investigating software development life-cycle and seeking for solution ideas to some common software development problems is a worthy topic to be chosen as our senior year project.

One of the greatest challenges of software development is gaining confidence and getting used to big and complex repositories. This is even more frustrating for junior developers. Adjusting to a new code base can be tiring when the developers don’t know anyone to whom they can consult and ask questions. Furthermore, even if there exist people to whome juniors can ask questions to, those people may not be the experts of the given code base. Consulting to people who are not the experts of the given code base can result in even more harm than it causes good. Thus, we believe that finding the correct expert for a code base is vital. In this report, the proposed project regards these concerns and comes up with a renovative and innovative solution.
